# Live Demo Runbook — "Streaks" in V0 (slide 10, ~20 min)

**Rule: type the prompt, then TALK while it generates (~30–60s each). Never stand silent watching a spinner.**

## Prep (before class, non-negotiable)

- Run this whole runbook once in V0 that afternoon.
- Keep that finished project open in a second tab = your backup.
  If live V0 dies: "here's one I made earlier" — nobody minds.

## Prompt 1 — describe the whole app (paste exactly)

> Build me a small web app called Streaks — a habit tracker.
> 3 sections: Today (my habits as a checklist), Add Habit (a name
> field and an emoji picker), and Streaks (each habit's current
> streak count). Bright playful style, one bold accent colour,
> big satisfying checkmarks.

**While generating:** "Notice I didn't say 'make me a habit app'. I gave it the blueprint — 3 named sections, what's in each, and a style line. That's why we write the blueprint BEFORE touching V0."

**When done:** click around. Tick a habit. Show the room it works.

## Prompt 2 — improve one thing

> Make the checkmark animate and turn green when I tap it.

**While generating:** "One change per prompt. Small steps. If I'd asked for five things and one broke, I wouldn't know which."

**When done:** tick a habit, let the animation land. Milk it.

## Prompt 3 — THE BUG BEAT (this is the review/fix lesson)

Go to Add Habit, add one — **ask the room to shout a habit name**. Then check the Today list. Very often it won't appear there (or find ANY real flaw — something will be off; there always is).

**Say:** "See this? I added it, and Today doesn't show it. The AI was confident, the app LOOKS done, and it's wrong. This is the whole 'you still judge the work' rule — I only caught it by TESTING."

If nothing is broken: "Normally something's broken here — you got a lucky run. Doesn't change the rule: test after every prompt."

**Fix prompt:**

> When I add a new habit in Add Habit, it should appear in the Today list immediately.

## Prompt 4 — one delight feature

> Add a flame icon next to any habit that has a streak of 3 days or more.

**While generating:** "This is the rhythm — describe, test, fix, improve. You'll do this exact loop yourself in an hour."

## Step 5 — publish

Click Publish in V0, copy the link, open it in a **new tab** — or on your phone, held up.

"That's a real URL. Anyone here can open this right now. That's the finish line tonight."

---

## Pacing note (learned the hard way, first run 2026-07-06)

**The stations are the class.** When you hit a timer slide — Idea Lock, the Guillotine, First Build — say the setup lines, start the timer, then *stop presenting*. Walk the room. Silence with a room full of people typing is the class working, not you failing. Only advance slides when a timer ends and the pacing takes care of itself.
